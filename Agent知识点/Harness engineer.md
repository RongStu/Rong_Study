## 认识 harness engineer
harness engineer 定义：为llm 和 agent 提供 完整的运行环境和控制系统

agent = model + harness

- model：用于思考解决问题、生成内容（类比野马）
- harness：管控系统，用于约束、引导、纠错、连接工具、管理上下文（像 “缰绳 + 马车”）

作用：让强大但不稳定的 AI 模型，变得可靠、可控、可用于工业生产。
组成：上下文管理、工具编排、验证机制与硬约束、状态管理、可观测性、人类接管与生命周期管理。

#### 1、 上下文管理（context engineer）
 - 项目指令文件：claude.md / agent.md agent启动时读取的基座文件
 - 动态上下文注入：从日志、指标、追踪文件实现实时信息管理
 - 上下文隔离：通过subagent作为上下文防火墙，隔离窗口独立运行
 - 上下文压缩；当上下文窗口被填满时，自动丢弃部分不重要信息，将主要内容压缩成简要的梗概

#### 2、工具编排
- 精选工具集：工具不是越多越好，精心策划，移除冗余选项
- mcp协议集成：标准化工具接口——文件系统、API、外部服务统一协议
- 沙箱隔离：执行环境与生产环境隔离

#### 3、验证机制
- 确定性约束：Linter、结构性测试、Pre-commit
- 生成评估分离：独立评估器
- 自动审查循环：多agent审查、迭代至满意

#### 4、状态管理
- llm 都是从无状态开始，需要设置外部化记忆：追踪记忆文件、结构化清单；增量git提交、检查点与恢复机制

#### 5、可观测性
- 执行追踪、质量分级、异常检测、反馈归因（根据失败模式，追溯到Harness缺陷，推动持续改进）
